# Project Instructions

## SVG Icons

- Do not add new inline SVG markup directly inside HTML files.
- When a new SVG icon is needed, create it as a separate `.svg` file under `assets/icons/`.
- Reference SVG icon files from HTML with an `<img>` element.
- Use descriptive, kebab-case file names such as `blog-book.svg` or `chevron-down.svg`.
- Use empty `alt=""` and `aria-hidden="true"` for decorative icons. Use meaningful Korean `alt` text when the icon conveys content.
- Reuse existing files in `assets/icons/` before creating a new icon.
