Legacy 소스 현황

  1. 서비스 진행에 필요한 모든 데이터를 앱 구동 시 한꺼번에 얻어오도록 되어 있음

    1. 초기 골프장에 납품되었던 태블릿이 wifi 전용 모듈이라 라운드 시작 전 무조건 모든 데이터를 가져와야함

  2. 모든 데이터가 Club 클래스의 하위 클래스로 정의되어 있어 데이터 분리가 필요함

  3. 사용하지 않는 데이터들이 많이 존재하며 이에 대한 정리가 필요함

  4. UI, 비지니스 로직, 데이터 처리등이 하나의 소스 (ex. MainActivity)에 구현되어 있어 분리가 필요함

  5. 데이터 fetch 등 비동기 처리 방식이 Handler (Legacy) 방식으로 되어 있어 가독성이 많이 떨어짐

설계

Clean Architecture 구조를 참고하여 캐디스코어 구조에 맞는 아키텍처 설계가 필요

모듈 단위 분리

  1. 하나의 소스파일에 UI, 비지니스 로직, 데이터 처리등 복잡하게 구현되어 있는 기존 소스 구조에서 (레이어, 기능) 단위 의존성을 제거하기 위해 모듈 단위로 나누어 구조를 설계합니다.

  2. 추후, 기능 단위 별로 담당자가 할당이 된다면 feature 단위로 모듈을 더 세분화 할 지 검토합니다.

아래와 같이 (core, app, data, domain, presentation)의 5개 모듈로 분리합니다.


App 모듈은 모든 Legacy 소스가 구현되어 있는 그 자체이며 그대로 현재 소스를 그대로 유지합니다.

구현 (Refactoring)

새롭게 앱을 만들어가는 것이 아닌 기존 Legacy 소스를 새롭게 정의한 아키텍처 구조에 맞게 Refactoring 해 나가는 방식으로 진행합니다.

작업 방향성

기존에 유기적으로 잘 동작하는 소스 구조에 영향을 주지 않고 작업을 진행할 수 있는 방향을 찾습니다.

  1. 기존 Legacy 소스를 변경하지 않고 Clean Architecture를 기반으로 새롭게 추가된 모듈에 동일한 기능을 새로 구현하는 방식으로 진행합니다.

  2. Refactoring 작업이 완료가 되면 충분한 테스트 기간을 확보하여 테스트를 진행합니다.

    1. 1개월 이상의 개발자 테스트

    2. 2개월 이상의 품질 테스트

  3. 충분한 테스트를 완료 후 새로운 아키텍처 기반의 소스로 프로덕션 배포를 진행합니다.

  4. 시장 품질 이슈까지 완벽히 검증이 되면 기존 Legacy 소스를 제거하는 작업을 진행합니다.

소스관리

기존 Legacy 에서 사용하던 pakage name 정리, (라이브러리, 개발언어) 등을 되도록 Google 에서 공식적으로 지원하는 최신 기술을 최대한 이용하도록 진행합니다.

개발언어

새롭게 작성하는 소스는 무조건 kotlin 으로 작성합니다.

SourceSet

기존 Legacy 소스와 새롭게 구현하는 소스를 구분하기 위해

  1. App 모듈 (Legacy) 에서는 별도의 SourceSet (kotlin)을 정의하여 폴더를 분리 합니다.

  2. 새롭게 추가되는 모듈에서는 무조건 Kotlin을 사용합니다.

Package 이름

새롭게 추가되는 모듈의 package 이름 (namespace) 은 아래의 이름 규칙에 따라 정의합니다.
com.smartscore.caddiescore.{모듈이름}

라이브러리

  1. AAC (android architecture component) 라이브러리를 적극 활용합니다.

  2. Hilt(DI) 라이브러리를 적용하여 모듈간의 dependency 를 연결해 줍니다.

Gradle

빌드 스크립트 언어로 kotlin DSL 을 적극 활용합니다.

Code Style

  1. 안드로이드 스튜디오에서 기본으로 제공하는 Kotlin code style을 사용합니다.

  2. Kotlin 소스 반영 전 Option+Command+L 버튼을 눌러 code style이 적용되도록 합니다.

데이터 정리

1차 Refactoring 작업 진행시에는 영향 범위가 적고 처리량이 적은 데이터를 선정하여 작업을 진행합니다.

  1. 서버에서 내려주는 JSON 문서로 먼저 정리하고 (default 값 등도 정의)

  2. 서버에서 LiveUrl 정보를 받아오는 네트워크 API를 구현합니다.(retrofit 라이브러리 이용)
